## 系統概述

### 🎯 系統目的
本系統用於處理固定式太陽能板的採集數據，從原始CSV檔案到最終的數據視覺化分析，提供完整的數據處理管線。

### 📦 系統組成
本系統包含以下6個主要程式：

| 程式名稱 | 功能說明 | 輸入 | 輸出 |
|---------|---------|------|------|
| `convert name1.py` | 檔案重命名 | 原始CSV檔案 | 重命名後的CSV檔案 |
| `power calculation2.py` | 功率計算 | 重命名後的CSV | 添加功率欄位的CSV |
| `power summary3.py` | 發電量匯總 | 功率計算後的CSV | 匯總報表 |
| `data preprocessing4.py` | 數據預處理 | 匯總後的CSV | 完整處理的CSV |
| `combine data 5.py` | 數據合併 | 多個處理後的CSV | 合併後的單一CSV |
| `fixed_panel_data_visualization.py` | 數據視覺化 | 合併後的CSV | 圖表分析 |

### 🔄 處理流程圖

```
原始數據資料夾
    │
    ├─ 子資料夾1/
    ├─ 子資料夾2/
    └─ 子資料夾3/
         │
         ▼
    【步驟1: 檔案重命名】convert name1.py
         │
         ▼
    已重命名/
    ├─ 傾角20度方位角180度.csv
    ├─ 傾角30度方位角160度.csv

         │
         ▼
    【步驟2: 功率計算】power calculation2.py
         │
         ▼
    已重命名/ (添加 Power(W) 和 Daily_Energy(Wh) 欄位)
         │
         ▼
    【步驟3: 發電量匯總】power summary3.py
         │
         ▼
    已重命名/
    └─ power_summary_YYYYMMDD_HHMMSS.csv
         │
         ▼
    【步驟4: 數據預處理】data preprocessing4.py
         │

         │
         ▼

         │
         ▼
    【步驟6: 數據視覺化】fixed_panel_data_visualization.py
         │
         ▼
    📊 圖表分析界面
```

## 處理流程說明

### 步驟1️⃣: 檔案重命名

#### 📝 功能說明
將原始檔名（如 `Z3A0412097(20241227150146).csv`）轉換為有意義的檔名（如 `傾角20度方位角180度1.csv`）

#### 🔧 產品代碼對應表

本程式內建了30個太陽能板的產品代碼對應關係：

| 產品代碼 | 傾角 | 方位角 | 類型 | 備註 |
|---------|-----|--------|------|------|
| Z3A0412097 | 20° | 180° | 固定式 | 編號1 |
| Z3A0412118 | 20° | 180° | 固定式 | 編號2 |
| Z3A0412115 | 30° | 180° | 固定式 | 編號1 |
| ... | ... | ... | ... | ... |
| Z3A0412111 | 25° | 0° | 追日系統 | 上方 |
| Z3A0512124 | 25° | 0° | 追日系統 | 下方 |

> 💡 **提示**: 如有新的太陽能板，需要在 `convert name1.py` 的 `product_mapping` 字典中添加對應關係

#### 📤 輸出結果
- 重命名後的檔案會複製到 `已重命名/` 子資料夾
- 原始檔案保持不變

---

### 步驟2️⃣: 功率計算

#### 📝 功能說明
為每筆數據添加功率計算和每日發電量統計

#### 🧮 計算公式

**功率計算**:
```
Power(W) = Voltage(V) × Current(A)
```

**每日發電量計算**:
```
Daily_Energy(Wh) = Σ(每小時整點的功率值)
```

> ⚠️ **注意**: 只使用整點數據（分鐘 = 0）來計算每日發電量

#### 📤 輸出結果
在原始CSV中添加兩個新欄位：
- `Power(W)` - 即時功率
- `Daily_Energy(Wh)` - 當日累計發電量

---

### 步驟3️⃣: 發電量匯總

#### 📝 功能說明
創建所有面板的發電量比較匯總表

#### 📊 匯總內容
- 自動識別相同角度組合的面板對
- 按日期排序顯示每對面板的發電量
- 計算兩個面板的發電量差異

#### 📤 輸出結果
生成 `power_summary_YYYYMMDD_HHMMSS.csv` 包含:
- 日期
- 面板1發電量
- 面板2發電量
- 發電量差異

---

### 步驟4️⃣: 數據預處理

#### 📝 功能說明
這是最複雜的步驟，包含多項數據處理任務：

1. **建立SQLite數據庫** - 便於數據管理
2. **導入CSV數據** - 讀取所有重命名後的CSV
3. **計算太陽位置** - 使用pvlib計算太陽方位角和天頂角
4. **整合照度數據** - （如有照度感測器數據）
5. **數據過濾** - 移除太陽位於地平線以下的數據
6. **生成15分鐘平均數據** - 將10分鐘數據轉換為15分鐘平均

#### 🌍 地理位置設定
```python
# 台灣地理位置（嘉義）
latitude = 23.5  # 緯度
longitude = 120.5  # 經度
altitude = 50  # 海拔高度(公尺)
timezone = 'Asia/Taipei'
```

#### 📤 輸出結果
生成 `complete_solar_data.csv` 包含完整欄位：
- `date` - 日期
- `hour_decimal` - 小時(十進位)
- `panel_id` - 面板ID
- `tilt_angle` - 傾角
- `azimuth_angle` - 方位角
- `power_W` - 功率(瓦)
- `solar_azimuth` - 太陽方位角
- `solar_zenith` - 太陽天頂角
- `illumination` - 照度（如有）

---

### 步驟5️⃣: 數據合併

#### 📝 功能說明
合併多個時段的 `complete_solar_data.csv` 成為單一檔案

#### 🔄 使用場景
當您有多個時間段的數據需要一起分析時：
```
20250301_1214/已重命名/complete_solar_data.csv
20251221_1231/已重命名/complete_solar_data.csv
20260101_0110/已重命名/complete_solar_data.csv
20260111_0120/已重命名/complete_solar_data.csv
```

#### 📤 輸出結果
生成 `combined_solar_data_20250301_20260120.csv`
- 包含所有時段的數據
- 自動去除重複記錄
- 按日期時間排序

---

### 步驟6️⃣: 數據視覺化

## 視覺化工具使用

### 🎨 啟動視覺化工具

```bash
python fixed_panel_data_visualization.py
```

### 📊 主要功能

#### 1️⃣ 年份選擇
- 視窗頂部的「選擇要分析的年份」框架
- 點擊年份按鈕（如「2025年」、「2026年」）切換年份
- 月份選項會自動更新為該年份的可用月份

#### 2️⃣ 月份選擇
- 在年份下方的「選擇要分析的月份」框架
- 點擊月份按鈕查看該月數據
- 支持的月份會根據數據自動顯示

#### 3️⃣ 三個分析頁面

**📅 日期比較頁面**
- 選擇多個日期進行發電曲線比較
- 顯示各日期的每小時發電量變化
- 疊加照度數據曲線
- 可調整圖例位置
- 快捷按鈕：全選、取消全選、選擇前5天、選擇後5天

**🔆 面板比較頁面**
- 選擇特定日期
- 比較多個面板在同一天的發電表現
- 可按傾角和方位角篩選面板
- 支持選擇多個面板同時比較
- 快捷按鈕：全選面板、取消全選、選擇前10個

**📈 月度統計頁面**
- 每日總發電量統計
- 面板總發電量比較
- 面板平均發電量比較
- 可選擇按數值排序
- 可選擇是否顯示數值標籤

### 💡 使用技巧

1. **快速查看特定月份數據**
   - 先選年份，再選月份，所有圖表會自動更新

2. **比較不同面板效能**
   - 使用「面板比較」頁面
   - 選擇同一天，勾選要比較的面板
   - 使用角度篩選快速找到相同配置的面板

3. **分析天氣影響**
   - 使用「日期比較」頁面
   - 選擇晴天和陰天進行對比
   - 觀察發電曲線的差異

4. **月度報表生成**
   - 使用「月度統計」頁面
   - 選擇「每日總發電量」查看整月趨勢
   - 可截圖保存作為報表

### ❓ Q2: 檔案重命名時提示「找不到產品代碼」

**解決方法**:
1. 開啟 `convert name1.py`
2. 找到 `product_mapping` 字典
3. 添加您的產品代碼對應關係：
   ```python
   'Z3A0XXXXXX': {'tilt': 20, 'azimuth': 180, 'type': 'normal', 'number': 1},
   ```

### ❓ Q3: 數據預處理很慢，可以加速嗎？

**A:** 數據預處理需要計算太陽位置，計算量較大

**優化建議**:
- 確保只處理必要的時間範圍
- 可以先處理小範圍數據測試
- 考慮使用SSD硬碟

### ❓ Q4: 視覺化工具顯示不出中文字體

**A:** matplotlib可能找不到中文字體

**解決方法**:
檔案已經設定好字體順序：
```python
plt.rcParams['font.family'] = ['DFKai-SB', 'Microsoft JhengHei', 'SimHei']
```

如仍有問題，可以手動指定系統中已安裝的中文字體。

### ❓ Q5: 合併數據時記憶體不足

**A:** 數據量太大超出可用記憶體

**解決方法**:
1. 分批次合併，不要一次合併太多檔案
2. 先合併部分檔案，再逐步添加
3. 升級電腦記憶體

### ❓ Q6: 找不到照度數據

**A:** 您的系統可能沒有照度感測器

**解決方法**:
- 這不影響主要功能
- 程式會自動使用發電量數據替代
- 視覺化工具仍可正常使用
